# 强制启用飞书工作流用量上报

## 目标

公司版 `sy-feishu-connect` 内置统一的飞书工作流地址和 Bearer Token。用户首次配置时只填写中文姓名和飞书工号，不接触统计地址或凭证；每次使用后自动上报当天累计使用次数。

## 配置行为

- 在 Go 配置模块中定义固定工作流 URL、固定 Bearer Token 和默认项目名 `sy-feishu-connect`。
- `config.toml` 和环境变量中的同名值不再影响运行时上报目标，避免用户漏配或改错。
- 保留已有 TOML 字段的解析能力，确保旧配置文件仍能正常加载；CLI 和图形向导新生成的配置不再写入 URL、Token 或项目名。
- 中文姓名保持必填，飞书工号保持可选。运行时优先使用飞书接口返回的姓名和工号，其次使用首次配置值。

## 上报行为

每次记录任务或命令后，程序向内置工作流发送 JSON：

```json
{
  "用户姓名": "段成亮",
  "飞书工号": "sy4044",
  "日期": "2026-07-10",
  "当日使用次数": 1,
  "项目": "sy-feishu-connect"
}
```

请求使用 `Authorization: Bearer <内置 Token>` 和 `Content-Type: application/json`。上报继续异步执行；网络错误或非 2xx 响应只写日志，不阻断 Codex 回复。

## 兼容与更新

- 新用户按教程执行全局安装、配置和启动。
- 已安装用户重新执行同一条 `npm install -g` 命令即可覆盖旧版本；已有姓名、工号、App ID 和 App Secret 配置继续保留，无需重新配置。
- 更新后重启正在运行的机器人进程，新版本和固定统计配置才会生效。

## 测试

- 配置测试证明固定工作流 URL、Token 和项目名会覆盖 TOML 与环境变量。
- 现有 HTTP 测试继续证明请求头、字段、日期和当日累计次数正确。
- CLI 和图形向导检查证明新配置文件不再包含工作流凭证字段。
- 完整运行 Go 测试、Node 语法检查、Python 编译检查、npm 打包检查和 Git 空白检查。

## 安全边界

工作流 URL 和 Token 将进入公开 Git 历史，任何能读取仓库的人都可能构造请求。飞书工作流应配置字段校验、频率限制和异常告警，并预留 Token 轮换方案。
