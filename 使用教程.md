# sy-feishu-connect 小白使用教程

这篇教程适合第一次使用的人。你只需要照着做，就能把飞书机器人接到本地 Codex CLI，然后在飞书里让 Codex 操作你的项目。

## 这个工具是干什么的

简单说：

```text
你在飞书发消息 -> 飞书机器人收到 -> sy-feishu-connect 转给本机 Codex -> Codex 操作项目 -> 结果回到飞书
```

它的好处是：不需要公网 IP，不需要域名，不需要配置 Webhook 服务器。飞书用长连接把消息推给你的电脑或服务器。

## 一、你需要先准备这些东西

### 1. 一台能运行 Codex 的电脑或服务器

在终端里确认 Codex 能正常打开：

```bash
codex --version
codex
```

如果这一步都不能用，先把 Codex CLI 安装、登录、配置好。

### 2. 一个要让 Codex 操作的项目目录

比如你的项目在：

```text
/Users/you/code/my-project
```

后面配置里的 `work_dir` 就填这个路径。

### 3. 一个飞书账号

个人账号或企业账号都可以。你需要能进入飞书开放平台创建「企业自建应用」。

## 二、最简单的启动方式：双击

下载仓库后，先打开完整的 `sy-feishu-connect` 文件夹。根目录里有几个最重要的双击入口：

```text
macOS：
双击打开配置工具.command
双击启动机器人.command

Windows：
双击打开配置工具.bat
双击启动机器人.bat
```

第一次使用时，先双击项目文件夹里的配置工具：

```text
macOS：双击打开配置工具.command
Windows：双击打开配置工具.bat
```

它会在本机启动一个网页配置向导，并自动打开浏览器页面。它会帮你检查环境、编译程序、生成 `config.toml`。页面里的「安装目录」和「Codex 项目目录」右边都有 `...` 按钮，点击后可以直接选择文件夹，不需要自己手打路径。

注意：不要只把双击入口单独拷贝到桌面或其它目录运行。这个文件旁边需要有 `setup-gui.py`。如果你看到类似下面的报错：

```text
can't open file '/Users/admin/sy/code/setup-gui.py': [Errno 2] No such file or directory
```

说明你双击的是放错位置的入口文件。正确做法是进入完整项目目录，比如：

```text
/Users/admin/sy/code/sy-feishu-connect/
```

然后双击这个目录里面的：

```text
macOS：双击打开配置工具.command
Windows：双击打开配置工具.bat
```

以后要启动机器人时，双击：

```text
macOS：双击启动机器人.command
Windows：双击启动机器人.bat
```

启动后会打开一个终端窗口。不要关闭这个窗口，关闭以后机器人就停止了。

如果你是开发者，也可以用命令行：

```bash
make build
./bin/sy-feishu-codex -config config.toml
```

Windows 命令行可以这样：

```bat
go build -o bin\sy-feishu-codex.exe .\cmd\sy-feishu-codex
bin\sy-feishu-codex.exe -config config.toml
```

能看到日志滚动，就说明服务正在运行。

## 三、创建飞书机器人应用

打开飞书开放平台：

https://open.feishu.cn/

然后按这个顺序点：

1. 进入「开发者后台」
2. 点击「创建企业自建应用」
3. 应用名称填：`Codex 助手`
4. 应用描述填：`飞书里操作 Codex CLI`
5. 创建应用

创建完成后，进入这个应用的后台。

## 四、打开机器人能力

在应用后台左侧找到：

```text
应用能力 -> 机器人
```

然后：

1. 点击启用机器人
2. 机器人名称可以填：`Codex助手`
3. 头像随便选一个
4. 保存

## 五、复制 App ID 和 App Secret

在应用后台左侧找到：

```text
凭据与基础信息
```

你会看到：

```text
App ID
App Secret
```

大概长这样：

```text
App ID:     cli_xxxxxxxxxxxxx
App Secret: xxxxxxxxxxxxxxxxxxxxx
```

先复制出来，后面要填到配置文件里。

注意：`App Secret` 不要发给别人。

## 六、配置飞书权限

在应用后台左侧找到：

```text
权限管理
```

搜索并添加这些权限：

```text
im:message.p2p_msg:readonly
im:message.group_at_msg:readonly
im:message:send_as_bot
```

如果你想让机器人识别群成员、处理 @某人，可以再加一个：

```text
im:chat.members:read
```

添加完成后，记得点发布或申请发布。飞书后台改权限后，不发布通常不会生效。

## 七、配置事件和回调

在应用后台左侧找到：

```text
事件与回调
```

### 1. 事件配置

进入「事件配置」：

1. 订阅方式选择「使用长连接接收事件」
2. 添加事件：`im.message.receive_v1`
3. 保存

### 2. 回调配置

进入「回调配置」：

1. 订阅方式选择「使用长连接接收事件」
2. 添加回调：`card.action.trigger`
3. 保存

`card.action.trigger` 是给飞书卡片按钮用的。如果不配置，有些按钮可能点了没反应。

## 八、发布应用

在应用后台左侧找到：

```text
版本管理与发布
```

创建一个新版本并发布。

如果你是企业账号，可能需要管理员审核。审核通过后，机器人才能正常使用。

## 九、写配置文件

回到终端，在项目目录里执行：

```bash
cp config.example.toml config.toml
```

打开配置文件：

```bash
vim config.toml
```

如果你不会用 `vim`，也可以用任何文本编辑器打开这个文件：

```text
config.toml
```

把里面改成类似这样：

```toml
[feishu]
app_id = "cli_xxxxxxxxxxxxx"
app_secret = "xxxxxxxxxxxxxxxxxxxxx"
domain = "feishu"
require_mention = true
allow_users = "*"
allow_chats = "*"
working_emoji = "OnIt"
done_emoji = "DONE"

[codex]
work_dir = "/Users/you/code/my-project"
cli_path = "codex"
model = ""
reasoning_effort = ""
mode = "suggest"
codex_home = ""
turn_timeout = "30m"

[bridge]
data_dir = "./data"
queue_messages = true
max_reply_chars = 3500

[log]
level = "info"
```

你必须改这 3 个地方：

```text
work_dir    改成你的项目目录
app_id      改成飞书应用的 App ID
app_secret  改成飞书应用的 App Secret
```

例子：

```toml
work_dir = "/Users/admin/code/my-web-app"
app_id = "cli_a123456789"
app_secret = "AbCdEfGh123456"
```

## 十、启动服务

最简单方式：双击根目录里的：

```text
macOS：双击启动机器人.command
Windows：双击启动机器人.bat
```

开发者也可以在项目目录里执行：

```bash
./bin/sy-feishu-codex -config config.toml
```

看到类似信息就说明启动成功：

```text
platform started project=my-project platform=feishu
sy-feishu-codex is running
```

这个终端窗口不要关。关掉以后机器人就收不到消息了。

## 十一、在飞书里测试

### 单聊测试

在飞书里搜索你的机器人，比如 `Codex助手`，打开单聊，发送：

```text
你好，帮我看一下这个项目是做什么的
```

如果配置没问题，Codex 会开始分析你的 `work_dir` 目录，然后把结果发回飞书。

### 群聊测试

把机器人拉进群，然后在群里发：

```text
@Codex助手 帮我检查一下项目结构
```

群聊里建议一定要 @机器人，这样飞书会明确把消息发给机器人。

## 十二、飞书底部自定义栏怎么配

这个是可选的，不配置也能用。

如果你想让用户在机器人单聊底部看到快捷按钮，可以去飞书应用后台：

```text
应用能力 -> 机器人 -> 机器人自定义菜单
```

推荐按 4 组配置：

| 分组 | 菜单项 |
| --- | --- |
| 会话 | 新建会话 `/new`、会话列表 `/sessions`、当前会话 `/status` |
| 执行 | 停止执行 `/stop`、当前状态 `/status`、工作目录 `/pwd` |
| 设置 | 模式 `/mode`、模型 `/model`、帮助 `/help` |
| 显示 | 显示思考 `/display full`、关闭思考 `/display compact`、极简模式 `/display quiet` |

当前版本已支持 `/help`、`/new`、`/status`、`/sessions`、`/reset`。其余菜单是推荐预留项，用来先把产品入口设计好；后续可以继续补齐命令能力。如果你只想放当前已支持的命令，就先配置 `/help`、`/new`、`/status`、`/sessions`。

如果你选「事件推送」，需要在飞书后台订阅对应的菜单事件；不熟悉的话，先用跳转链接就行。

在飞书里可以发：

```text

```


## 十五、常见问题

### 1. 飞书里发消息没有反应

按顺序检查：

1. 运行 `sy-feishu-connect` 的终端有没有关。
2. 飞书应用有没有发布。
3. 事件 `im.message.receive_v1` 有没有添加。
4. 订阅方式是不是「长连接」。
5. 机器人有没有被添加到单聊或群聊。
6. 群聊里有没有 @机器人。

### 2. 提示 App ID 或 App Secret 错误

回到飞书后台「凭据与基础信息」，重新复制 `App ID` 和 `App Secret`。

注意不要多复制空格。

### 3. Codex 不工作

先在本机终端直接运行：

```bash
codex
```

如果 Codex 本身打不开，先修 Codex。

### 4. 卡片按钮点了没反应

检查「事件与回调」里有没有添加：

```text
card.action.trigger
```

如果暂时不想配卡片，可以在配置里改成：

```toml
enable_feishu_card = false
```

### 5. 想让它一直后台运行

可以安装为后台服务：

```bash
./bin/sy-feishu-codex -config config.toml
```

看日志：

```bash
# 日志会直接显示在当前终端
```

重启：

```bash
# 停止后重新运行：./bin/sy-feishu-codex -config config.toml
```

停止：

```bash
# 在运行窗口按 Ctrl+C 停止
```

## 最后检查清单

启动前确认这几项：

- Codex CLI 本机能正常运行。
- `work_dir` 是真实存在的项目路径。
- 飞书应用已经启用机器人。
- 飞书权限已经添加并发布。
- 事件 `im.message.receive_v1` 已添加。
- 回调 `card.action.trigger` 已添加。
- `app_id` 和 `app_secret` 已填入配置。
- `sy-feishu-connect` 正在运行。

这些都对了，基本就能用了。
