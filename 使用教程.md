# sy-feishu-connect 小白使用教程

这份教程给第一次使用的人看。目标很简单：让你在飞书里发一句话，本机 Codex 就能去看或改你的项目。

## 先看懂它

```text
你在飞书发消息
  -> 飞书机器人收到
  -> sy-feishu-connect 转给本机 Codex
  -> Codex 在你的项目目录里执行
  -> 结果回到飞书
```

它不需要公网 IP，不需要域名，也不需要自己写 Webhook 服务器。

## 第一步：安装

打开终端，复制这一行执行：

```bash
npm install -g https://github.com/snake-mustang/sy-feishu-connect/archive/refs/heads/main.tar.gz
```

如果提示没有 `npm`，先安装 Node.js。安装好以后再运行上面的命令。

以后如果发布到 npm 官方 registry，可以改用更短的：

```bash
npm install -g sy-feishu-connect
```

## 第二步：检查是否装好

继续复制这一行：

```bash
sy-feishu-connect doctor
```

你希望看到类似：

```text
✅ Node.js
✅ Codex CLI
✅ sy-feishu-connect core
```

如果 Codex CLI 是 `❌`，先在本机安装并登录 Codex，再回来检查。

## 第三步：生成配置

复制这一行：

```bash
sy-feishu-connect setup
```

它会问你几个问题，照着填就行。

| 问题 | 怎么填 |
| --- | --- |
| 配置文件保存在哪里 | 不懂就直接回车，用默认值 |
| Codex 要操作的项目目录 | 你的真实代码项目路径，比如 `/Users/you/code/my-app` |
| 飞书 App ID | 从飞书后台「凭据与基础信息」复制 |
| 飞书 App Secret | 从飞书后台「凭据与基础信息」复制 |
| 你的姓名 | 用来统计是谁在用，可以填真实姓名 |
| 你的工号 | 用来统计是谁在用，可以填工号 |
| 统计上报地址 | 没有就直接回车，只保存在本机 |

默认配置会保存到：

```text
~/.sy-feishu-connect/config.toml
```

App ID 和 App Secret 会写到这个文件的 `[feishu]` 里：

```toml
[feishu]
app_id = "cli_xxxxxxxxxxxxx"
app_secret = "xxxxxxxxxxxxxxxxxxxxx"
domain = "feishu"
```

如果你填错了，可以重新运行 `sy-feishu-connect setup`，也可以手动打开这个文件改这两行。

## 第四步：飞书后台必须手动做的 5 件事

打开 [飞书开发者后台](https://open.feishu.cn/app)。

### 1. 创建企业自建应用

1. 点击「创建企业自建应用」。
2. 名称建议填 `Codex 助手`。
3. 描述可以填 `飞书里操作 Codex CLI`。

### 2. 启用机器人

路径：

```text
应用能力 -> 机器人
```

启用机器人，设置名称和头像。

### 3. 复制 App ID 和 App Secret

路径：

```text
凭据与基础信息
```

复制：

```text
App ID
App Secret
```

回到 `sy-feishu-connect setup` 里填进去。

### 4. 添加权限和事件

权限管理里搜索并添加下面这些权限。

更简单的方式：在「权限管理」点击「批量处理」->「批量导入」，直接粘贴：

![飞书权限批量导入示意图](./docs/assets/feishu-permission-bulk-import.png)

```json
{
  "scopes": {
    "tenant": [
      "contact:user.base:readonly",
      "im:message.group_at_msg:readonly",
      "im:message.p2p_msg:readonly",
      "im:message.group_msg",
      "im:message:send_as_bot"
    ],
    "user": []
  }
}
```

`im:message.group_msg` 是敏感权限。如果你只想让群聊 @ 机器人时触发，可以删掉这一行后再导入。

必选权限：

| 权限名称 | 权限标识 | 用途 |
| --- | --- | --- |
| 读取用户发给机器人的单聊消息 | `im:message.p2p_msg:readonly` | 接收私聊消息 |
| 获取群组中用户 @ 机器人消息 | `im:message.group_at_msg:readonly` | 接收群聊 @ 消息 |
| 以应用身份发送群消息 | `im:message:send_as_bot` | 发送回复 |

可选权限：

| 权限名称 | 权限标识 | 用途 |
| --- | --- | --- |
| 获取与更新用户基本信息 | `contact:user.base:readonly` | 自动对应姓名/工号 |
| 获取群组中所有消息 | `im:message.group_msg` | 敏感权限；仅关闭 @ 要求时需要 |
| 添加消息表情回复 | `im:message:reaction` | 处理中/完成表情 |

如果你是第一次配置，只加必选权限就够了。`im:message.group_msg` 是敏感权限，默认不需要。

事件与回调里选择：

```text
使用长连接接收事件
```

订阅事件：

| 事件名称 | 事件标识 | 用途 |
| --- | --- | --- |
| 接收消息 | `im.message.receive_v1` | 接收用户发送给机器人的消息 |

改完权限和事件以后，一定要去「版本管理与发布」发布应用。没发布通常不会生效。

### 5. 配置底部自定义栏

路径：

```text
应用能力 -> 机器人 -> 机器人自定义菜单
```

推荐配置 4 组：

| 分组 | 菜单 |
| --- | --- |
| 会话 | 新建会话 `/new`、会话列表 `/sessions`、当前会话 `/status` |
| 执行 | 停止执行 `/stop`、当前状态 `/status`、工作目录 `/pwd` |
| 设置 | 模式 `/mode`、模型 `/model`、帮助 `/help` |
| 显示 | 显示思考 `/display full`、关闭思考 `/display compact`、极简模式 `/display quiet` |

当前版本已支持 `/help`、`/new`、`/status`、`/sessions`、`/stats`、`/whoami`、`/reset`。其余菜单可以先作为预留入口。

## 第五步：启动机器人

飞书后台配置完成后，终端执行：

```bash
sy-feishu-connect start
```

这个终端窗口不要关闭。关闭以后，机器人就停止了。

## 第六步：在飞书里测试

单聊机器人，发送：

```text
你好，帮我看一下这个项目是做什么的
```

群聊里发送：

```text
@Codex助手 帮我检查一下项目结构
```

群聊默认要 @机器人。

## 怎么统计是谁用的

有两层统计：

### 本机统计

服务会自动保存：

```text
data/usage_events.jsonl
data/usage_summary.json
```

飞书里发送：

```text
/stats
```

可以看总次数、成功失败、用户 Top。

让每个用户发送：

```text
/whoami
```

可以看到自己的飞书用户标识，方便你把 `open_id` 对应到真实姓名。

如果你在飞书后台已经导入并发布了 `contact:user.base:readonly` 权限，系统会自动尝试把 `open_id` 解析成姓名/工号。成功后 `/stats` 会显示类似：

```text
张三/E001 (ou_xxx)
```

如果还是只看到 `ou_xxx`，一般是权限没发布、应用没有通讯录可见范围，或者飞书接口暂时没返回。这个不影响 Codex 使用，后面用 `/whoami` 的结果人工对应也可以。

### 远程统计

如果你想集中统计所有安装者，请准备一个 HTTP 接收地址，比如：

```text
n8n webhook
你自己的 API
serverless function
日志收集服务
```

然后在 `sy-feishu-connect setup` 的「统计上报地址」里填这个 URL。

之后每次使用都会向这个地址发送一条 JSON。里面会包含安装者姓名/工号、飞书用户标识、飞书姓名/工号、命令、是否成功、耗时等字段。

普通用户不需要推 GitHub，也推不到你的 GitHub main。统计应该走日志接口。

## 常见问题

### sy-feishu-connect doctor 失败

先看哪一项是 `❌`。

- Node.js 失败：安装 Node.js。
- Codex CLI 失败：安装并登录 Codex。
- sy-feishu-connect core 失败：安装 Go 后重新运行 `sy-feishu-connect doctor`。

### 飞书里发消息没有反应

按顺序检查：

1. `sy-feishu-connect start` 的终端窗口还开着吗？
2. 飞书应用发布了吗？
3. 事件 `im.message.receive_v1` 添加了吗？
4. 订阅方式是不是长连接？
5. 机器人添加到单聊或群聊了吗？
6. 群聊里有没有 @机器人？

### Codex 操作的是哪个目录

就是 `sy-feishu-connect setup` 里填写的「Codex 要操作的项目目录」。

它应该是你的业务项目，比如：

```text
/Users/you/code/my-web-app
```

不要填 `sy-feishu-connect` 自己的安装目录，除非你就是想让 Codex 修改这个工具。

## 最后检查清单

- `npm install -g https://github.com/snake-mustang/sy-feishu-connect/archive/refs/heads/main.tar.gz` 已执行。
- `sy-feishu-connect doctor` 全部是 `✅`。
- `sy-feishu-connect setup` 已生成配置。
- 飞书应用已启用机器人。
- 权限和事件已添加并发布。
- 底部自定义栏已按 4 组配置。
- `sy-feishu-connect start` 正在运行。
- 飞书里测试消息能收到回复。
