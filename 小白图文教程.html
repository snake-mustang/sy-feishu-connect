从 0 到 1：用飞书操作本地 Codex

你只需要做三件事：第一次双击配置工具；去飞书后台完成权限和机器人菜单；以后每天双击启动机器人。

第一次双击配置检查 Codex、Git/Go/Make、下载源码、编译、生成配置文件。
必须手动做创建飞书应用、启用机器人、添加权限、发布应用、配置底部菜单。
以后双击启动双击“启动机器人”后保持窗口打开，飞书就能调用 Codex。

0. 先用一张图看懂它

核心理解：飞书只是入口，真正执行任务的是你本机的 Codex CLI。
这里原本可以放飞书对话截图。为了保证用户下载仓库后离线打开不缺图，本教程主要使用内置 SVG 示意图。

1. 第一次：双击配置工具

先打开完整的 sy-feishu-connect 文件夹。macOS 双击 双击打开配置工具.command，Windows 双击 双击打开配置工具.bat。第一次使用先双击配置工具，它会启动本地网页配置向导，并帮用户做这些事：

检查

检查 codex、git、go 是否可用；有 make 就用，没有也能用 go build。

下载和编译

自动 clone 或更新 sy-feishu-connect，并编译出可执行文件。

生成配置

目录右侧有 ... 按钮可直接选择文件夹，再填写 App ID、App Secret，自动写入 config.toml。

运行方式

macOS：双击仓库根目录里的 双击打开配置工具.command
Windows：双击仓库根目录里的 双击打开配置工具.bat
浏览器会自动打开：http://127.0.0.1:随机端口/
目录怎么选：安装目录是 sy-feishu-connect 工具自己的目录，通常不用改；Codex 要操作的项目目录 是你希望飞书里的 Codex 去查看、修改、运行的业务项目，例如 /Users/you/code/my-web-app。不要选 sy-feishu-connect，除非你就是想让 Codex 修改这个工具本身。
不要把双击入口单独拖到桌面或上一层目录运行。它旁边需要有 setup-gui.py。如果看到 can't open file ... setup-gui.py，说明双击入口放错位置了，请进入完整的 sy-feishu-connect 文件夹再双击。
工具不能替你登录飞书后台，也不能替你点击飞书权限。飞书后台的配置还是要手动做。

2. 飞书后台必须手动做的 5 件事

创建企业自建应用打开 飞书开发者后台，创建企业自建应用，名称建议填 Codex 助手。
启用机器人打开 飞书开发者后台，进入你的应用，路径：应用后台 ->「应用能力」->「机器人」。启用后设置头像和名称。
复制 App ID 和 App Secret打开 飞书开发者后台，进入你的应用，路径：应用后台 ->「凭据与基础信息」。这两个值要填进配置工具。
添加权限、事件和回调打开 飞书开发者后台，进入你的应用，配置权限管理、事件与回调；订阅方式选长连接，改完后记得发布应用。
配置底部自定义栏打开 飞书开发者后台，进入你的应用，路径：应用能力 -> 机器人 -> 机器人自定义菜单。推荐配置 4 个菜单组。

3. 权限、事件、回调照着填

权限管理里添加

权限为什么要它
im:message.p2p_msg:readonly接收单聊消息。
im:message.group_at_msg:readonly接收群聊 @机器人消息。
im:message:send_as_bot机器人把 Codex 结果发回飞书。
im:chat.members:read可选，用于群成员识别和 @某人。

事件与回调里添加

事件配置

订阅方式：使用长连接接收事件

im.message.receive_v1

回调配置

订阅方式：使用长连接接收事件

card.action.trigger
飞书后台改完权限、事件、回调以后，一定要去「版本管理与发布」发布新版本。

4. 配置文件只需要看懂 3 个值

如果你用了半自动工具，它会帮你生成这个配置。如果手动写，大概长这样：

[feishu]
app_id = "cli_xxxxxxxxxxxxx"
app_secret = "xxxxxxxxxxxxxxxxxxxxx"
domain = "feishu"
require_mention = true
allow_users = "*"
allow_chats = "*"
working_emoji = "OnIt"
done_emoji = "DONE"

[codex]
work_dir = "/Users/you/code/my-project"
cli_path = "codex"
model = ""
reasoning_effort = ""
mode = "suggest"
codex_home = ""
turn_timeout = "30m"

[bridge]
data_dir = "./data"
queue_messages = true
max_reply_chars = 3500

[log]
level = "info"

5. 以后：双击启动机器人

小白启动

macOS：双击启动机器人.command
Windows：双击启动机器人.bat

启动后会出现终端窗口，不要关闭它。

命令行启动

macOS/Linux:
./bin/sy-feishu-codex -config config.toml

Windows:
bin\sy-feishu-codex.exe -config config.toml

适合开发者排查问题。

在飞书里发什么

你好，帮我看一下这个项目是做什么的
@Codex助手 帮我检查一下最近一次提交

6. 底部自定义栏推荐配置 4 个

这是给用户看的快捷菜单。路径一般是：飞书应用后台 ->「应用能力」->「机器人」->「机器人自定义菜单」。

1. 会话

  • 新建会话：/new
  • 会话列表：/sessions
  • 当前会话：/status

2. 执行

  • 停止执行：/stop
  • 当前状态：/status
  • 工作目录：/pwd

3. 设置

  • 模式：/mode
  • 模型：/model
  • 帮助：/help

4. 显示

  • 显示思考：/display full
  • 关闭思考：/display compact
  • 极简模式：/display quiet
如果飞书后台不支持多级菜单，就把 4 个作为一级菜单；如果支持多级菜单，就按上面的分组放二级按钮。
